Аутентификация и авторизация

Кто ты и что тебе можно делать в системе

Идентификация

Процесс определения пользователя по уникальному идентификатору (логин, email, ID). Первый шаг перед аутентификацией.

Базовый процесс

Аутентификация

Проверка подлинности пользователя (пароль, токен, биометрия). Подтверждает, что ты — это ты.

Базовый процесс

Авторизация

Определение прав доступа после аутентификации. Решает, что пользователь может делать в системе.

Базовый процесс

OAuth 2.0

Стандарт делегирования доступа. Позволяет приложению получить токен без пароля пользователя.

Протокол

OpenID Connect (OIDC)

Слой идентификации на базе OAuth 2.0 с JWT-токенами для подтверждения личности.

Протокол

JWT (JSON Web Token)

Компактный токен для передачи утверждений (claims). Используется в OIDC и API-аутентификации.

Токен

SAML 2.0

XML-стандарт для обмена данными аутентификации между Identity Provider и Service Provider.

Протокол

MFA (Multi-Factor Authentication)

Многофакторная аутентификация: комбинация знания, владения и биометрии для повышения безопасности.

Механизм

2FA (Two-Factor Authentication)

Двухфакторная аутентификация: пароль + одноразовый код (OTP) или биометрия.

Механизм

SSO (Single Sign-On)

Единый вход: один логин даёт доступ ко всем связанным системам без повторной аутентификации.

Концепция

Kerberos

Протокол на билетах (tickets) для сетевой аутентификации. Стандарт в Active Directory.

Протокол

RBAC (Role-Based Access Control)

Контроль доступа на основе ролей: пользователи получают права по своей роли в системе.

Авторизация

ABAC (Attribute-Based Access Control)

Авторизация на основе атрибутов (время, локация, устройство). Более гибкий, чем RBAC.

Авторизация

PBAC (Policy-Based Access Control)

Авторизация по политикам: динамические правила на основе контекста (риск, поведение).

Авторизация

FIDO2

Стандарт для беспарольной аутентификации: WebAuthn + CTAP для аппаратных ключей и биометрии.

Беспарольная

Passkeys

Актуальный тренд 2025: синхронизированные крипто-ключи для MFA без паролей (на базе FIDO2).

Беспарольная

Biometric Authentication

Биометрическая аутентификация: отпечатки пальцев, Face ID, голос. Фактор "владения" в MFA.

Биометрия

OTP (One-Time Password)

Одноразовый пароль: генерируется для каждой сессии (TOTP/HOTP в MFA).

Токен

Certificate-Based Authentication

Аутентификация по цифровым сертификатам (X.509). Используется в PKI и VPN.

Сертификаты

Federated Identity

Федеративная идентичность: SSO через внешние провайдеры (Google, Microsoft).

Федерация

SCIM (System for Cross-domain Identity Management)

Стандарт автоматизации управления идентичностями в облаке (проведение пользователей).

Управление

Identity Provider (IdP)

Провайдер идентичности: сервис, управляющий аутентификацией (Okta, Azure AD).

Роль

Service Provider (SP)

Провайдер сервиса: приложение, запрашивающее доступ (SAML/OAuth).

Роль

2SV (Two-Step Verification)

Двухшаговая верификация: упрощённая MFA без полного многофакторного подхода.

Механизм

ITDR (Identity Threat Detection and Response)

Обнаружение и реагирование на угрозы идентичности (Indeed ITDR 2.0, 2025).

Мониторинг

Passwordless Authentication

Беспарольная аутентификация: FIDO2, магические ссылки, аппаратные ключи.

Беспарольная

Hardware Token

Аппаратный токен: YubiKey, смарт-карты для MFA и PKI-аутентификации.

Устройство

Continuous Authentication

Непрерывная аутентификация: мониторинг поведения в реальном времени (2025 тренд).
